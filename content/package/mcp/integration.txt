# MCP & Memory Integration Guide

## Overview

This guide explains how MCP (Model Context Protocol) and Memory work together in indusagi to create a powerful, context-aware agent system.

## Architecture Overview

```
┌─────────────────────────────────────────────────────────┐
│                   Agent Application                       │
└────────────┬────────────────────────────┬────────────────┘
             │                            │
      ┌──────▼─────────┐         ┌───────▼──────────┐
      │  MCP Integration│         │ Memory System    │
      │  ├─ Tools       │         │ ├─ Storage      │
      │  ├─ Resources   │         │ ├─ Embeddings   │
      │  └─ Services    │         │ ├─ Processors   │
      └──────┬─────────┘         │ └─ Vector Store │
             │                    └───────┬──────────┘
      ┌──────▼──────────────────────────▼─────┐
      │     Unified Tool & Context System       │
      │  - Tool Execution                      │
      │  - Context Retrieval                   │
      │  - Decision Making                     │
      └──────────────────────────────────────┘
             │
      ┌──────▼─────────────────────────┐
      │    External Services & Data     │
      │ ├─ APIs (MCP servers)          │
      │ ├─ Databases (MCP resources)   │
      │ └─ User Context (Memory)       │
      └────────────────────────────────┘
```

## Integration Scenarios

### Scenario 1: Context-Aware Tool Execution

**Problem**: Tools can work better if they know about past decisions and user preferences.

**Solution**: Combine MCP tools with Memory context.

```typescript
import { initializeMCP } from "indusagi/mcp";
import { Memory, InMemoryStorage, InMemoryVectorStore, createOpenAIEmbedder } from "indusagi/memory";
import { Agent, ToolRegistry } from "indusagi/agent";

// Setup
const toolRegistry = new ToolRegistry();
const { pool, toolCount } = await initializeMCP(toolRegistry, process.cwd());

// Setup Memory
const memory = new Memory({
  storage: new InMemoryStorage(),
  vector: new InMemoryVectorStore(),
  embedder: createOpenAIEmbedder({
    apiKey: process.env.OPENAI_API_KEY,
  }),
  options: {
    semanticRecall: { enabled: true, topK: 5 },
    workingMemory: { enabled: true },
    lastMessages: 10,
  },
});

// Create agent with both
const agent = new Agent({
  toolRegistry,
  model: "gpt-5.4",
});

// Execute with context
const threadId = "user-123";
await memory.createThread(threadId);

// Get memory context
const context = await memory.getContext({ threadId });

// Pass to agent
const response = await agent.run({
  message: "Create a new API endpoint",
  context: context.workingMemory, // Remember previous preferences
  tools: toolRegistry.getAll(),    // Use MCP tools
});

// Store result in memory for future use
await memory.addMessage({
  threadId,
  role: "assistant",
  content: response.content,
  type: "text",
});
```

### Scenario 2: Tool Selection Based on Context

**Problem**: Different users have different tool preferences and workflows.

**Solution**: Use memory to guide tool selection.

```typescript
async function selectTools(threadId: string, task: string) {
  // Get user's working memory (preferences)
  const workingMemory = await memory.getWorkingMemory(threadId);
  
  // Semantic search for similar past tasks
  const similarTasks = await memory.searchSemantic(task, 3);
  
  // Select tools based on context
  const availableTools = toolRegistry.getAll();
  
  if (workingMemory.preferredTools) {
    // Prioritize tools user prefers
    return availableTools.filter(t => 
      workingMemory.preferredTools.includes(t.name)
    );
  }
  
  if (similarTasks.length > 0) {
    // Use tools from similar past tasks
    const toolsUsed = similarTasks
      .flatMap(t => t.metadata.toolsUsed)
      .filter(Boolean);
    return availableTools.filter(t => toolsUsed.includes(t.name));
  }
  
  // Default: all tools
  return availableTools;
}
```

### Scenario 3: Multi-Step Workflows with Memory

**Problem**: Complex workflows need to remember decisions across multiple steps.

**Solution**: Use MCP tools with Memory-backed state.

```typescript
async function executeWorkflow(threadId: string, goal: string) {
  const steps = [];
  
  // Step 1: Plan using memory context
  const context = await memory.getContext({ threadId });
  const plan = await agent.think({
    goal,
    context: context.workingMemory,
  });
  
  // Step 2: Execute tools from MCP servers
  for (const step of plan.steps) {
    const result = await executeToolFromMCP(step.tool, step.params);
    
    // Step 3: Store intermediate results in memory
    await memory.addMessage({
      threadId,
      role: "assistant",
      content: `Executed ${step.tool}: ${result}`,
    });
    
    steps.push({ tool: step.tool, result, timestamp: new Date() });
  }
  
  // Step 4: Reflect and store summary
  const summary = await agent.reflect({ steps, goal });
  
  // Update working memory with summary
  const workingMemory = await memory.getWorkingMemory(threadId);
  workingMemory.lastWorkflow = summary;
  await memory.updateWorkingMemory({
    threadId,
    updates: workingMemory,
  });
  
  return summary;
}
```

## Usage Patterns

### Pattern 1: Agent with Tool Discovery

Automatically discover and register MCP tools, then use agent:

```typescript
import { initializeMCP } from "indusagi/mcp";

const registry = new ToolRegistry();
const { pool, toolCount } = await initializeMCP(registry);

const agent = new Agent({
  toolRegistry: registry,
  model: "gpt-5.4",
});

console.log(`Agent has ${toolCount} tools available`);
```

### Pattern 2: Memory-Aware Assistant

Create an assistant that learns from conversations:

```typescript
const memory = new Memory({
  storage: new InMemoryStorage(),
  vector: new InMemoryVectorStore(),
  embedder: createOpenAIEmbedder({
    apiKey: process.env.OPENAI_API_KEY,
  }),
  options: {
    semanticRecall: { enabled: true },
    workingMemory: { enabled: true },
  },
});

class MemoryAwareAssistant {
  async chat(threadId: string, message: string) {
    // Retrieve memory context
    const context = await memory.getContext({ threadId });
    
    // Generate response with context
    const response = await this.agent.run({
      message,
      context,
    });
    
    // Remember the interaction
    await memory.addMessage({
      threadId,
      role: "user",
      content: message,
    });
    await memory.addMessage({
      threadId,
      role: "assistant",
      content: response,
    });
    
    return response;
  }
}
```

### Pattern 3: Context-Aware Tool Execution

Execute tools with relevant context injected:

```typescript
class ContextualToolExecutor {
  async executeWithContext(threadId: string, toolName: string, params: any) {
    // Get recent memory context
    const context = await memory.getContext({ threadId });
    
    // Enhance tool parameters with context
    const enhancedParams = {
      ...params,
      context: context.workingMemory,
      recentDecisions: context.semanticMatches,
    };
    
    // Execute tool
    const tool = toolRegistry.get(toolName);
    const result = await tool.execute(enhancedParams);
    
    // Store execution in memory
    await memory.addMessage({
      threadId,
      role: "assistant",
      content: `Used ${toolName}: ${result}`,
    });
    
    return result;
  }
}
```

## Data Flow Examples

### Example 1: User Starts a Project

```
1. User: "Start a TypeScript project"
   ↓
2. Agent queries Memory: "What's the user's preference?"
   Memory returns: "User prefers TypeScript, functional patterns, strict config"
   ↓
3. Agent selects tools from MCP: filesystem, git
   ↓
4. Agent executes tools:
   - filesystem/create_file: tsconfig.json (with strict settings)
   - git/init: Initialize repository
   ↓
5. Agent stores in Memory:
   "Created TypeScript project with functional patterns"
   ↓
6. Next time user starts project → Agent remembers preferences
```

### Example 2: User Asks for Help on Existing Project

```
1. User: "How should I structure this API?"
   ↓
2. Agent queries Memory:
   - Semantic search: "Similar API discussions"
   - Working memory: "Current project goals"
   ↓
3. Results show past decision: "Used REST with JWT auth"
   ↓
4. Agent uses MCP tools:
   - filesystem/read: Current API structure
   - github/search: Similar repos
   ↓
5. Agent provides contextualized recommendation:
   "Given your project goals (from memory) and current structure,
    I recommend extending the existing REST pattern..."
   ↓
6. Stores new information in Memory for future reference
```

### Example 3: Multi-Tool Workflow

```
1. User: "Set up CI/CD for our project"
   ↓
2. Agent gets Memory context:
   - Current tech stack (from working memory)
   - Past CI/CD setup patterns (semantic recall)
   ↓
3. Agent creates execution plan:
   - Use GitHub MCP: create workflow file
   - Use filesystem MCP: add config files
   - Store MCP: save configuration
   ↓
4. Execute tools in sequence:
   Tool 1: github/create_file → CI workflow
   Tool 2: filesystem/write → local configs
   Tool 3: store/update → database config
   ↓
5. Store in Memory:
   - Execution steps
   - Decisions made
   - Lessons learned
   ↓
6. Future setups use this pattern
```

## Best Practices

### 1. Always Create Thread First

```typescript
// DON'T: Assume thread exists
await memory.addMessage({ threadId: "user-1", ... });

// DO: Create thread if needed
await memory.createThread("user-1");
await memory.addMessage({ threadId: "user-1", ... });
```

### 2. Use Scoped Memory for Multi-User Apps

```typescript
// Track memory per user
const memory = new Memory({
  options: {
    workingMemory: { scope: "resource" }, // Per-resource scope
    semanticRecall: { scope: "resource" },
  },
});

// Each user has isolated memory
await memory.addMessage({
  threadId: "user-123",
  resourceId: "project-456", // Scoped to project
  role: "user",
  content: "...",
});
```

### 3. Update Working Memory Deliberately

```typescript
// DON'T: Store everything
// DO: Store only important context
const workingMemory = await memory.getWorkingMemory(threadId);
workingMemory.userPreferences = {
  framework: "typescript",
  styling: "tailwind",
  database: "postgresql",
};

await memory.updateWorkingMemory({
  threadId,
  updates: workingMemory,
});
```

### 4. Handle Tool Failures with Memory

```typescript
try {
  const result = await executeTool(toolName, params);
} catch (error) {
  // Remember the failure
  await memory.addMessage({
    threadId,
    role: "assistant",
    content: `Tool ${toolName} failed: ${error.message}`,
  });
  
  // Try alternative approach
  // or escalate to user
}
```

## Performance Considerations

### Memory + MCP Optimization

```typescript
// 1. Batch semantic searches
const queries = ["preference", "similar tasks", "past decisions"];
const results = await Promise.all(
  queries.map(q => memory.searchSemantic(q, 3))
);

// 2. Cache tool list from MCP
const toolCache = new Map();
for (const client of pool.getAllClients()) {
  const tools = await client.listTools();
  toolCache.set(client.serverName, tools);
}

// 3. Limit memory retrieval
const context = await memory.getContext({
  threadId,
  options: {
    maxMessages: 20,    // Limit history
    similarityThreshold: 0.8, // Be selective
  },
});
```

### Connection Pooling

```typescript
// Share pool across requests
const pool = new MCPClientPool({ servers });
await pool.connectAll();

// Reuse in request handler
app.post("/execute-tool", async (req, res) => {
  const { tool, params } = req.body;
  
  // Pool is already connected, just use it
  const client = pool.getClient("filesystem");
  const result = await client.callTool(tool, params);
  
  res.json(result);
});

// Keep pool for lifecycle
process.on("exit", () => pool.disconnectAll());
```

## Troubleshooting

### MCP Tools Not Accessible in Memory

Check that tool metadata is stored with message:

```typescript
await memory.addMessage({
  threadId,
  role: "assistant",
  content: result,
  metadata: {
    toolsUsed: ["filesystem/read", "github/search"],
    executionTime: 234, // ms
  },
});
```

### Memory Context Too Large

Reduce context window:

```typescript
const context = await memory.getContext({
  threadId,
  options: {
    maxMessages: 10,         // Fewer messages
    maxSemanticResults: 3,   // Fewer semantic matches
    workingMemoryOnly: true, // Skip history if needed
  },
});
```

### Slow Semantic Search

Increase threshold to get fewer, more relevant results:

```typescript
const config = {
  semanticRecall: {
    enabled: true,
    threshold: 0.85, // More selective
    topK: 3,         // Fewer results
  },
};
```
